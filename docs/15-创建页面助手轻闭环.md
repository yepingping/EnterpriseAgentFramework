# 创建页面助手轻闭环

本文记录“创建页面助手”v1 的当前落地边界。它面向项目详情页中的推荐入口，帮助用户从前端页面动作目录生成 Agent Studio 草稿。

## 当前已落地

- 项目详情页的“创建页面助手”进入项目级向导页面。
- 向导读取当前项目的前端页面目录、页面动作目录、可选 API 资产和 LLM 模型实例。
- 页面已接入但没有动作时，用户可以复制最小 `pageBridge.registerAction` 模板，也可以在平台手工声明页面动作草案。
- 手工声明写入现有 `eaf_page_registry` 与 `eaf_page_action_registry`，不新增表；`metadata_json.source` 标记为 `MANUAL_DRAFT`。
- 草稿生成复用 `/api/agent/studio/generate-draft`，请求中新增 `pageActions` 资源，后端会把匹配资源规范化为 `PAGE_ACTION` GraphSpec 节点。
- 草稿通过 `sessionStorage` 交给 `/agent/new/studio`，用户在 Studio 检查画布、参数映射和发布校验后再保存。
- Studio 支持新草稿直接保存：保存时创建 Agent，并跳转到新 Agent 的 Studio 页面。

## 接入边界

- v1 不引入 Cursor SDK、代码仓库 Agent 或自动 PR。
- v1 不承诺扫描任意前端页面并自动理解按钮语义；页面可以自动上报，动作语义必须来自 SDK 注册或平台手工声明。
- 手工声明动作可以先生成草稿，但真实运行仍需要业务前端补齐对应 handler。
- `PAGE_ACTION` 的运行、调试和发布校验继续沿用现有 Runtime、嵌入式会话和页面动作目录机制。

## 关键接口

- 页面目录：`GET /api/platform/embed/pages`
- 页面动作目录：`GET /api/platform/embed/page-actions/catalog`
- 手工声明动作：`POST /api/platform/embed/page-actions/catalog/manual`
- 草稿生成：`POST /api/agent/studio/generate-draft`

## 验证建议

- 后端优先跑 `LlmWorkflowDraftGeneratorTest` 与 `PlatformEmbedOpsControllerTest`。
- 前端优先跑 `ai-admin-front` 的 `npm run build`。
- 运行时验收需要打开已接入前端页面，确认页面 SDK 能轮询、执行并回传 `PAGE_ACTION` 调试结果。
